Better binary tree visualization

There are tons of binary tree algorithm scripts out there. We love binary trees. They are a classic introduction to recursion in programming, capable of a unique problem solving ability. When it comes down to visualising binary trees, a simple diagram will work for educational purposes. But what about other purposes?
When I googled binary trees I would get something like this:

When you look into some trees that were actually generated with code, you can find something like this:

Looks awesome, but it's so straight and unnatural. It's missing so much tree character. I want a binary tree to look like an actual tree. I've searched the web on a qusest to find the best looking binary tree out there.. There were close to no binary trees that looked like actual trees. What a bummer.

I wanted to fix that, so I introduced more random() into the generation, with some colors. Basically, I've made an algorithm that will recursively draw circles while moving up, slightly changing directions and getting smaller over time. To add more unpredictability, I've added chances to the branch that it will split after a certain length. Also, after the ongoing branch reaches a certain length, there is a chance that it will grow a leaf. Here is the result:

Some come out better, some worse. All and all I'm really glad how the generator turned out. Took a while to find the sweetspot in the configuration.

Every once in a while you get a stump ~

You can play around with the generation live on the main page of my website. I've also posted the full code on github. The github code has a UI with different settings that you can play around with.
Btw, watching the leaves grow is very mesmerizing.
